用迭代编程

2019-09-16

写作本文的起因是群友在群里问了，画板能否化简二次根式。花了点时间想出了算法（数论无力）

结果先贴这里

x=ab\sqrt{x}=a\sqrt{b}x​=ab​，其中 a 要最大整数，b 为整数，即 b 为最小整数。枚举 b 使得 x/b\sqrt{x/b}x/b​ 是整数

分类讨论

{b=xx=bb=1x=ab≠1∧b≠xx=ab\begin{cases} b=x & \sqrt{x}=\sqrt{b}\\ b=1 & \sqrt{x}=a\\ b≠1∧b≠x& \sqrt{x}=a\sqrt{b}\\ \end{cases} ⎩⎪⎨⎪⎧​b=xb=1b​=1∧b​=x​x​=b​x​=ax​=ab​​

所以重点是 b ，即题目本质是找 Square Free 因子

然后 O(n) 枚举（不是最高效的，但是素因子分解画板无力（可以搞，但是枚举就够了））。

接下来描述如何枚举

  1. 建变量 b=1
  2. m1=b+1
  3. 构造点 A(b,0)
  4. b→b+1 的迭代
  5. 判断 sqrt(x/b) 是否整数，是则停止迭代，并取迭代终点，度量 x 坐标

用 C/C++ 就是这么描述

int b=1;
for(;b<=x;b++)
{
    if(floor(sqrt(x/b))*floor(sqrt(x/b))*b==x)
    {
        break;
    }
}

用 Mathematica 这么描述

With[{x = 4}, NestWhile[# + 1 &, 1, !IntegerQ@Sqrt[x/#] &]]

附演示图

下面简单总结如何“编程”

画板编程有以下特点：

  1. 没有数组，只能得到返回值
  2. 要用参数作为迭代的对象，用点来反映参数的变化。用迭代停止条件作为判断。变量本身不能存。

第 2 点准确说是，结果是由迭代次数，迭代原象和迭代停止条件动态确定的。无法直接调用。只能通过附着在点上调用。

网络画板的嵌套测量目前没思路，先略。

总结：所以拿这玩意弄迭代。除非闲得 egg 疼

加上几何画板令人窒息的 sgn 嵌套，网络画板略微好些，但是效率感人。😞。

结论：除非真的需要（复分形），不然该干嘛干嘛去吧。用于研究可以，用于实践是真的不实用。

伪代码如下

int gcd(int a,int b)
{
    int m;
    while(m!=0)
    {
        m=mod(a,b);
        a=b;
        b=m;
    }
    return a;
}

然后改写程序

int gcd(int a,int b)
{
    while(true)
    {
        m=mod(a,b);
        a=if(m!=0,b,a);
        b=if(m!=0,m,b);
    }
    return b;
}
  1. 建参数 a，b
  2. 构造点 B(b,0)
  3. 计算 m1=mod(a,b)
  4. 计算 m2 = if(m!=0,b,a)
  5. 计算 m3 = if(m!=0,m,b)
  6. 迭代 a→m2,b→m3,次数 O(min⁡(a,b))O(\min(a,b))O(min(a,b))
